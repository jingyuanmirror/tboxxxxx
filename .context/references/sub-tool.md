## 1. 背景与目标

### 1.1 背景

当前首页 `MagicInput` 组件支持 6 种工具模式（PPT 演示、Web 搜索、Excel 分析、Code 生成、Mail 撰写、小红书），但每个模式对应的子功能均未拆解。用户选择工具后，只能靠自然语言描述任务意图，缺少**引导性的子任务入口**，导致使用门槛偏高、任务意图模糊。

### 1.2 目标

1. 为每个工具模式设计 3–6 个子功能，覆盖该模式最高频的任务场景。
2. 选中工具模式后，在输入框上方（或 Toolbar 区域）展开一排子功能 Chip，一键设定任务上下文。
3. 子功能选中后，自动填充 `textarea` 的 Placeholder/提示前缀，并可携带必要参数（输出格式、文件要求等）注入到 Prompt 上下文中。
4. 支持"做图"新工具模式，补齐图像创作场景。

## 4. 交互设计

### 4.1 触发逻辑

```
用户点击工具模式(三点菜单) → 选择顶级工具模式
   → 工具模式 Chip 出现在 Toolbar 区域
   → 同时在 Chip 行右侧展开「子功能选择条」
      → 子功能以 Pill/Badge 形式横向排列（最多 6 个，超出可横向滚动）
      → 用户点击子功能 Pill
         → 对应 Chip 高亮选中
         → textarea placeholder 切换为子功能 Prompt 前缀
         → 若子功能有必须上传文件，文件区域出现提示高亮
```

### 4.2 子功能展开区域布局

```
┌──────────────────────────────────────────────────────────────────────┐
│  textarea 区域（输入区不变）                                           │
├──────────────────────────────────────────────────────────────────────┤
│  [+]  [@]  [···模式]  ┆ [PPT 演示 ×]  ←  工具模式 Chip               │
│                         [🔧 旧PPT优化] [📊 调研报告] [🚀 路演模板] …  │
│                                ↑ 子功能选择条，仅在选中工具模式后出现   │
├──────────────────────────────────────────────────────────────────────┤
│  [Tbox ▾]    [→ 发送]                                                 │
└──────────────────────────────────────────────────────────────────────┘
```

### 4.3 状态管理

| 状态字段 | 类型 | 说明 |
|---------|------|------|
| `selectedMode` | `Mode \| null` | 当前选中的顶级工具模式 |
| `selectedSubTool` | `SubTool \| null` | 当前选中的子功能，切换顶级模式时重置 |
| `subToolPromptPrefix` | `string` | 注入到 textarea placeholder 的子功能前缀 |
| `subToolFileHint` | `string \| null` | 子功能建议上传的文件类型提示 |

### 4.4 子功能 Pill 样式规范

- **默认状态**：圆角边框，背景 `rgba(0,0,0,0.04)`，字号 12–13px，height 26px
- **选中状态**：蓝色背景 `#1a73e8`，白色文字，带 0.5px 投影
- **Hover 状态**：边框颜色加深，轻微背景变化
- **图标**：每个子功能使用对应的 SVG 图标（16×16），颜色继承文字色

### 4.5 动画规范

- 子功能条出现：`translateX(-8px) → 0` + `opacity 0 → 1`，时长 `180ms`，`ease-out`
- 切换顶级模式时：旧条淡出（`80ms`），新条淡入（`180ms`）
- Pill 选中：scale `1 → 1.04 → 1`，时长 `120ms`

## 5. 数据结构设计

### 5.1 类型定义（TypeScript）

```typescript
interface SubTool {
  key: string;                    // 唯一标识，如 "ppt_optimize"
  name: string;                   // 显示名称，如 "旧PPT一键优化"
  promptPrefix: string;           // 注入 textarea 的提示前缀
  fileHint?: string;              // 建议上传的文件类型，如 ".pptx"
  fileRequired?: boolean;         // 是否强制要求上传文件
  icon: () => React.ReactElement; // SVG 图标渲染函数
}

interface Mode {
  name: string;
  iconKey: string;
  isSpecial: boolean;
  icon: () => React.ReactElement;
  subTools: SubTool[];            // 新增：子功能列表
}
```

### 5.2 Prompt 上下文注入格式

当用户选择子功能后，向 AI 注入的系统上下文格式：

```json
{
  "tool_mode": "ppt",
  "sub_tool": "ppt_roadshow",
  "sub_tool_name": "路演报告",
  "user_input": "用户的自然语言输入",
  "uploaded_files": ["filename.pptx"],
  "model": "tbox"
}
```

## 8. 验收标准

1. 选择顶级工具模式后，500ms 内展示对应子功能选择条
2. 子功能 Pill 点击后，textarea placeholder 立即切换为对应前缀文案
3. 点击顶级模式的「×」关闭按钮，子功能选择条同时消失，`selectedSubTool` 重置
4. 切换顶级工具模式，子功能选择条更新为新模式的子功能列表
5. 子功能选择条在宽度不足时支持横向滚动，不撑开父容器
6. 所有子功能均有对应图标，图标渲染无错误
