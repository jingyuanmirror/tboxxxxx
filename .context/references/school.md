# PRD：Tbox 高校版首页改版

**文档版本**：v1.0  
**创建日期**：2026-03-12  
**状态**：草稿

---

## 一、背景与目标

### 1.1 背景

Tbox 当前产品定位为职场人 AI 工作助手，任务场景（商务邮件、竞品分析、路演 Deck 等）全面面向职业用户。在向高校推广时，学生打开首页会感知到明显的定位错位，核心痛点和使用场景均无法共鸣，导致转化率低、留存差。

### 1.2 目标

推出 **Tbox 高校版**，针对在校大学生、研究生群体进行首页功能与体验的专项改版，让学生在首次使用时即能感知到「这是为我设计的工具」，从而提升首页转化率和用户留存。

### 1.3 目标用户

| 细分群体 | 核心需求 |
|----------|----------|
| 大一 / 大二 | 课程笔记整理、课堂汇报 PPT、基础学习规划 |
| 大三 | 实习准备、简历、求职信、专业课课程论文 |
| 大四 / 应届生 | 毕业论文、求职面试、毕业答辩课件 |
| 硕士研究生 | 文献综述、学术论文写作、组会汇报、实验报告 |
| 博士研究生 | 英文学术写作、文献速读、论文润色、投稿准备 |

### 1.4 成功指标

- 新用户首次任务完成率（目标：>60%，较职场版提升 20%+）
- 次日留存率（目标：>40%）
- 核心功能（论文助手、文献速读）月活渗透率（目标：>30%）

## 二、产品整体设计思路

### 2.1 核心改变

```
职场版                     高校版
─────────────────────────────────────────
"你的 AI 工作搭子"    →    "你的 AI 学习伙伴"
职业身份（岗位）       →    学业身份（年级 + 专业 + 目标）
工作任务场景          →    学业任务场景
结果 = 工作输出        →    结果 = 学业成果
```

### 2.2 App Mode 扩展

在现有 `appMode: 'normal' | 'beginner' | 'openclaw'` 基础上，新增 `'student'` 模式。

```typescript
appMode: 'normal' | 'beginner' | 'openclaw' | 'student'
```

高校版作为独立模式存在，可通过切换入口或专属渠道链接（如高校合作落地页）激活。

## 三、Onboarding 流程改版

### 3.1 当前问题

现有 `ProfileIntroCard` 询问「你是做什么的」，职业选项（产品经理、运营、销售）对学生无效。

### 3.2 新版问卷设计（三步）

#### Step 1：年级选择

**问题**：你现在是几年级的学生？  
**选项**（单选）：

| 选项 | 标识符 |
|------|--------|
| 大一 | `freshman` |
| 大二 | `sophomore` |
| 大三 | `junior` |
| 大四（应届） | `senior` |
| 硕士研究生 | `master` |
| 博士研究生 | `phd` |

#### Step 2：专业方向选择

**问题**：你的专业大方向是？  
**选项**（单选）：

| 选项 | 标识符 |
|------|--------|
| 工科 / 理科 | `stem` |
| 文科 / 社科 | `humanities` |
| 商科 / 经管 | `business` |
| 医学 / 生命科学 | `medical` |
| 艺术 / 设计 | `arts` |
| 其他 | `other` |

#### Step 3：近期最想搞定的事

**问题**：最近最想搞定哪件事？  
**选项**（单选，内容根据前两步动态调整）：

- 通用选项：论文 / 作业、备考 / 期末、实习 / 求职、课堂汇报、学英语、其他
- **大三/大四/应届**额外展示：简历优化、面试准备
- **研究生/博士**额外展示：文献综述、英文论文润色、投稿准备

### 3.3 Greeting 文案逻辑

根据 grade × major 的组合，首屏 Greeting 个性化展示：

| 年级 | 专业 | Title | Subtitle |
|------|------|-------|----------|
| 大一/大二 | 任意 | `{name}，大学生活加油！` | 课堂汇报、作业笔记、学习规划，我来帮你搞定 |

| 硕士 | STEM | `{name}，科研路上有我` | 文献速读、实验报告、组会汇报 PPT，交给我 |
| 硕士 | 人文/商科 | `{name}，学术写作找我` | 论文框架、文献综述、学术改写，我是你的写作搭子 |
| 博士 | 任意 | `{name}，搞学术就找我` | 英文论文润色、文献精读、投稿 Cover Letter，随时可以开工 |

## 五、新增功能模块

### 5.1 学业 DDL 看板

**改造自**：现有 `ScheduledTasks` 组件  
**核心改变**：任务类型从「定时工作流」变为「学业截止日期管理」

**功能描述**：
- 支持添加：作业截止日期、考试时间、答辩日期、论文节点
- 展示：距截止日期的倒计时标签（3天内标红、7天内标黄）
- AI 辅助：点击任务可一键发起「帮我拆解这个任务的执行计划」
- 首页卡片：展示最近 3 个 DDL，点击进入完整看板

**DDL 卡片示意**：
```
┌─────────────────────────────────────┐
│ 📅 数据结构课程论文      ⚠️ 3天后   │
│ 操作系统期末考试        📌 12天后   │
│ 毕业论文初稿提交        🟡 21天后   │
└─────────────────────────────────────┘
```

### 5.3 「我的课程」知识库

**改造自**：现有 `KnowledgeView`  
**左侧导航栏入口**：由「知识库」改为「我的课程」

**功能改变**：
- 资料组织从「项目/话题」改为「课程」层级
- 支持上传课件 PDF / PPT，AI 自动提炼核心知识点列表
- 每门课程支持关联该课程的 DDL 任务
- 支持一键生成该课程的「期末复习提纲」

---

### 5.4 论文写作 Flow（P2，核心差异化功能）

**入口**：首页任务场景「写论文」→「开始毕业论文/学术论文写作」

**Flow 链路**：

```
Step 1 选题确认
  └─ AI 辅助评估题目可行性、建议研究范围
  
Step 2 文献检索
  └─ 自动搜索相关文献 → 提炼每篇摘要 → 生成文献引用列表

Step 3 论文提纲规划
  └─ 根据题目和文献，生成章节结构提案 → 用户确认/修改

Step 4 分章节写作
  └─ 每章单独对话，AI 结合前一章上下文继续写
  
Step 5 润色与格式
  └─ 全文语言润色 → 格式检查（字数/标题层级/引用格式）
```

**数据**：每个 Flow 进度自动保存，可中断恢复，保存在 `TopicSpace`（话题空间）中。

---

### 5.5 文献速读功能（P2）

**入口**：首页「查资料」场景 → 「上传文献速读」

**功能**：
- 输入方式：上传 PDF / 粘贴 DOI / 粘贴摘要链接
- 输出内容：
  - 研究问题与动机（1-2句）
  - 核心方法（2-3句）
  - 主要发现（3-5条 bullet）
  - 局限性与未来方向
  - 引用格式（支持 APA / MLA / GB/T 7714 切换）
- 可继续追问：「这篇文章的研究方法适合我用吗？」

## 九、技术实现要点

### 9.1 Profile 数据结构扩展

```typescript
interface StudentProfile {
  name?: string;
  grade: 'freshman' | 'sophomore' | 'junior' | 'senior' | 'master' | 'phd';
  major: 'stem' | 'humanities' | 'business' | 'medical' | 'arts' | 'other';
  primaryGoal: 'paper' | 'exam' | 'job' | 'presentation' | 'english' | 'other';
  school?: string; // 可选，院校名称

```

### 9.2 动态场景排序

在 `TaskLauncher` 中根据 `StudentProfile.grade` 动态生成 `TASK_SCENARIOS` 排序，逻辑封装为纯函数 `getStudentScenarios(profile: StudentProfile): TaskScenario[]`。

### 9.3 学期节点检测

```typescript
function getCurrentSemesterPhase(): SemesterPhase {
  const month = new Date().getMonth() + 1; // 1-12
  if ([2, 9].includes(month)) return 'opening';
  if ([4, 11].includes(month)) return 'midterm';
  if ([6, 1].includes(month)) return 'final';
  if ([8, 9, 10, 11].includes(month)) return 'autumn_recruit';
  if ([2, 3, 4].includes(month)) return 'spring_recruit';
  if ([3, 4, 5].includes(month)) return 'thesis';
  return 'normal';
}
```

## 十一、待决策事项

1. **是否独立部署**：高校版作为独立子域名（`edu.tbox.ai`）还是在主产品内通过 mode 切换？建议先 mode 内切换，验证数据后再考虑独立域名。
2. **高校合作方式**：是否需要院校管理员账号体系（班级/课程管理）？本期 PRD 暂不涉及，后续根据 BD 进展补充。
3. **文献检索数据源**：文献速读和文献检索功能需要对接哪些学术数据库（知网/万方/arXiv/Semantic Scholar）？需要提前确认 API 授权。
4. **AI 学术写作合规性**：产品需明确「AI 辅助写作」与「学术不端」边界，在适当位置加入免责说明与使用引导。
