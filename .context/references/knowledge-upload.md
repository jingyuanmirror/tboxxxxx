**文档版本**：v1.1

**状态**：评审中

## 一、背景与目标

### 1.1 背景

知识库由三大内容板块构成：**上传的内容**、平台制作的内容、存下来的模版。本文档聚焦第一板块——「上传的内容」，即用户从本地或第三方来源导入的文件/素材的管理功能。

### 1.2 目标

- 让用户能够快速上传、查找、整理本地文件
- 提供灵活的视图切换与多维度排序/筛选
- 通过「专辑」机制实现轻量级层级管理
- 保持操作链路简洁，上传至查看一步可达

### 1.3 成功指标

| 指标 | 目标值 |
|------|--------|
| 单次上传完成率 | ≥ 95% |
| 文件检索耗时（已有文件） | ≤ 5s |
| 用户 7 日留存（上传功能）| ≥ 60% |

## 四、详细功能说明

### 4.1 视图模式

#### 4.1.1 列表视图（List View）

- 每行展示一条文件记录，信息密度高，适合快速浏览大量文件
- 列字段（可拖拽调整顺序）：

| 列名 | 说明 | 默认显示 |
|------|------|----------|
| 文件图标 + 名称 | 点击可预览/打开 | ✅ |
| 文件格式 | PDF / DOCX / MP4 … | ✅ |
| 所属专辑 | 支持多专辑标签 | ✅ |
| 上传时间 | yyyy-MM-dd HH:mm | ✅ |
| 文件大小 | 自动换算 KB/MB/GB | ✅ |
| 操作 | 更多菜单（⋯） | ✅ |

- 支持多选（Checkbox）+ 批量操作

#### 4.1.2 方块视图（Grid View）

**核心展示逻辑：专辑优先 + 散列文件并排**

方块视图中，内容分为两类卡片混合展示：

| 卡片类型 | 展示条件 | 代表什么 |
|----------|----------|----------|
| 专辑卡片 | 该专辑内至少有 1 个文件 | 一组文件的集合 |
| 文件卡片 | 该文件**未归属任何专辑** | 独立的散列文件 |

> **规则：已归属专辑的文件不在顶层出现**。用户点击专辑卡片后，钻入专辑内部，以列表视图查看其中所有文件。

**排列规则**

- 默认顺序：专辑卡片靠前，散列文件卡片靠后，组内按上传时间降序
- 排序操作同时作用于两类卡片
- 搜索时：同时匹配专辑名称 + 文件名，均在顶层展示（不钻入）

**专辑卡片设计**

```
┌─────────────────────────────┐
│  [叠层纸张缩略图：最多展示  │
│   3 张子文件的内容预览线条] │
│                             │
│  市场研究                   │
│  12 个文件   02-25          │
└─────────────────────────────┘
```

- 缩略图区：3 张错位叠纸，每张纸右上角显示对应文件的格式色标，内部以内容线条模拟文字段落
- 悬停：显示「打开」+「更多操作」按钮
- 支持多选（Checkbox）

**文件卡片设计**

```
┌─────────────────────────────┐
│  [单张纸张缩略图：            │
│   内容预览线条 + 右上角       │
│   文件格式色标徽章]          │
│                             │
│  Q4 推广方案预算分配表       │
│  XLSX                 02-25 │
└─────────────────────────────┘
```

- 缩略图区：叠纸效果（三张），右上角文件格式色块徽章，内部线条模拟内容
- 悬停：显示「预览」+「更多操作」按钮
- 支持多选（Checkbox）

**钻入专辑内部（二级页面）**

- 点击专辑卡片 → 顶部面包屑切换为 `我的上传 > 专辑名`
- 内容区切换为该专辑内的文件**列表视图**（信息密度高，适合管理）
- 视图切换按钮依然可用（列表 / 方块两种）
- 专辑内的文件不再有"所属专辑"列（已在专辑内，无需重复显示）
- 顶部新增「编辑专辑」按钮，可修改专辑名称、描述

#### 4.1.3 视图切换

- 右上角切换按钮（列表图标 / 方块图标），状态持久化至用户偏好
- 列表视图：所有内容（专辑 + 文件）在同一平铺表格中共存，专辑行以文件夹图标区分，可展开查看子文件

### 4.2 排序与筛选

#### 4.2.1 排序

支持以下维度单字段排序，升/降序切换：

| 排序维度 | 说明 |
|----------|------|
| 上传时间 | 默认降序（最新在前） |
| 名称 | 按字母 / 拼音 A–Z |
| 文件格式 | 按格式名称字母序 |
| 文件大小 | 从大到小 / 从小到大 |

- 列表视图：点击列表头触发排序
- 方块视图：顶部「排序」下拉选择器

#### 4.2.2 筛选

支持多条件组合筛选，实时生效：

| 筛选维度 | 控件类型 | 可选值 |
|----------|----------|--------|
| 文件格式 | 多选 Checkbox | 图片 / 视频 / 音频 / 文档 / 压缩包 / 其他（动态来源于已有文件） |
| 所属专辑 | 树形多选 | 所有专辑（含未分类） |
| 上传时间 | 日期范围选择器 | 今天 / 本周 / 本月 / 自定义范围 |

- 筛选面板：点击工具栏「筛选」按钮展开；已激活筛选项在入口处以 Badge 数字提示
- 支持「一键清除全部筛选」

#### 4.2.3 搜索

- 顶部搜索框，支持按**文件名**关键词模糊搜索
- 搜索与筛选互不冲突，可叠加使用
- 无结果时显示空状态插画 + 「清除搜索条件」按钮

### 4.3 新建 / 上传功能

#### 4.3.1 入口

- 页面右上角主操作按钮：**「+ 新建」**（Primary Button）
- 点击后弹出下拉菜单：
  ```
  ┌──────────────────┐
  │  📁  新建专辑    │
  │  ⬆️  上传文件    │
  └──────────────────┘
  ```

#### 4.3.2 上传文件流程

**Step 1 — 触发上传**

- 点击「上传文件」或将文件拖拽至页面任意区域（Drop Zone 高亮提示）

**Step 2 — 文件选择**

- 弹出系统文件选择器，支持多文件同时选中
- 支持拖拽文件夹（自动展开为文件列表，文件夹名可选性地映射为专辑）

**Step 3 — 上传前配置（弹窗）**

```
┌─────────────────────────────────────────┐
│  上传文件                          [×]  │
├─────────────────────────────────────────┤
│  已选择 3 个文件                        │
│  ┌──────────┬────────┬────────────────┐ │
│  │ 文件名   │ 大小   │ 状态           │ │
│  ├──────────┼────────┼────────────────┤ │
│  │ 报告.pdf │ 2.3MB  │ 待上传         │ │
│  │ 图片.png │ 450KB  │ 待上传         │ │
│  │ 数据.xlsx│ 1.1MB  │ 待上传         │ │
│  └──────────┴────────┴────────────────┘ │
│                                         │
│  加入专辑（可选）  [选择专辑 ▼]         │
│                                         │
│           [取消]  [开始上传]            │
└─────────────────────────────────────────┘
```

**Step 4 — 上传中**

- 弹窗底部或右下角 Toast 显示整体进度条
- 每个文件单独显示上传进度百分比 + 剩余时间估算
- 支持取消单个文件或全部取消
- 上传失败的文件高亮显示错误原因 + 单独重试按钮

**Step 5 — 上传完成**

- 全部成功：Toast 提示「已成功上传 X 个文件」，列表自动刷新并滚动至新文件
- 部分失败：弹窗保留，高亮失败文件，提供「重试失败项」按钮

#### 4.3.3 新建专辑流程

1. 点击「新建专辑」→ 弹出轻量输入弹窗：专辑名称（必填）+ 描述（选填）
2. 支持选择父专辑（实现嵌套，最多 2 层）
3. 点击「创建」后专辑出现在左侧筛选面板与「所属专辑」选项中

| 操作 | 说明 |

### 4.5 预览设计

#### 4.5.1 设计哲学

预览不仅是"查看文件"，而是一次**沉浸式内容接触**。设计目标：

- **背景退场**：使用深色半透明遮罩（`bg-black/80 backdrop-blur-md`），将用户注意力完全聚焦于内容
- **内容主导**：预览区域占据视口 80%+ 的空间；文件名、格式、操作栏以最小化方式陈列在边缘
- **优雅过渡**：从卡片原位放大进入预览（scale + fade），而非突兀弹窗
- **单手可关**：点击遮罩区域、按 Esc 均可退出，不强迫用户找关闭按钮

#### 4.5.2 文件预览（单文件）

**触发方式**

- 方块视图：hover 时点击「预览」按钮
- 列表视图：点击文件名；或 hover 时点击「预览」按钮
- 键盘：选中文件后按 Space 键

**预览界面布局**

```
┌──────────────────────────────────────────────────────────────────┐
│  深色遮罩层（bg-black/80 backdrop-blur-md）                      │
│                                                                  │
│  ┌─── 顶部信息栏（高度 52px，透明底，渐隐效果）──────────────┐  │
│  │  ← 返回   [文件名]  [格式徽章]              [下载] [⋯] [×]│  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                  │
│                   ┌── 内容主体区 ───────────────────────────┐   │
│                   │                                         │   │
│                   │   文档：白底卡片 + 内容渲染              │   │
│                   │   图片：原始比例，超出则缩放              │   │
│                   │   视频：黑底播放器，居中                  │   │
│                   │   音频：专辑封面 + 波形 + 控制栏          │   │
│                   │   不支持格式：文件图标 + 下载提示         │   │
│                   │                                         │   │
│                   └─────────────────────────────────────────┘   │
│                                                                  │
│  ← 上一个                                          下一个 →     │
└──────────────────────────────────────────────────────────────────┘
```

**各格式渲染方式**

| 格式 | 渲染方式 | 设计细节 |
|------|----------|----------|
| PDF | iframe 或 PDF.js | 白底卡片，内容垂直滚动，分页明显 |
| DOCX / XLSX / PPTX | 转换后预览（后续接入） | v1 降级：显示格式图标 + 文件信息 + 下载按钮 |
| 图片 | `<img>` 原比例 | 支持鼠标滚轮缩放，双击复原 |
| 视频 | HTML5 `<video>` | 黑底全宽，自动 focus 控制栏 |
| 音频 | 自定义播放器 | 大号唱片封面占位图 + 时间轴 + 控制按钮 |
| MD / TXT | Markdown 渲染 | 白底卡片，`prose` 排版 |
| 其他 | 文件信息卡 | 格式图标 + 名称 + 大小 + 下载 CTA |

**导航**

- 同专辑内（或当前筛选结果内）支持左右切换（`←` `→`）
- 键盘左右方向键触发切换
- 显示「x / n」进度指示

#### 4.5.3 专辑预览（聚合视图）

专辑没有"单一内容"可展示，因此预览呈现的是一个**专辑内容摘要面板**，而非全屏预览器。

**触发方式**

- 方块视图：hover 专辑卡片 → 点击「预览」按钮
- 交互形式：从右侧滑入的侧边抽屉（而非全屏遮罩），保留背景内容的可见性，传递"窥视"而非"进入"的感觉

**侧边抽屉布局**

```
┌────────────────── 右侧抽屉（宽 360px，高 100vh）────────────────┐
│  专辑名称                                          [打开] [×]   │
│  ─────────────────────────────────────────────────────────────  │
│  12 个文件   最近更新 02-25                                      │
│                                                                  │
│  ┌── 文件缩略图网格（2列，最多展示 6 个，其余折叠）──────────┐  │
│  │  [图1] [图2]                                               │  │
│  │  [图3] [图4]                                               │  │
│  │  [图5] [图6]  +6 更多                                      │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                  │
│  格式分布                                                        │
│  ████░░  PDF 4  DOCX 3  XLSX 3  其他 2                          │
│                                                                  │
│  最近添加                                                        │
│  · Q4 推广方案预算分配表.xlsx         02-25                      │
│  · 芯片市场竞品分析报告.pdf           02-24                      │
│  · 2025 年产品路线图.pptx             02-23                      │
│                                                                  │
│           [ 打开专辑，查看全部文件 ]                             │
└──────────────────────────────────────────────────────────────────┘
```

**设计细节**

- 抽屉使用 `shadow-2xl` + 白色背景，与深色遮罩（`bg-black/20`）形成轻量层次感
- 文件缩略网格每张均可点击，直接跳入该文件的全屏预览
- 格式分布用色条（各格式对应品牌色）直观呈现文件结构
- 「打开专辑」按钮置底，作为强 CTA

## 六、异常状态处理

| 场景 | 处理方式 |
|------|----------|
| 文件格式不支持 | 上传前校验，弹窗内标红提示，不进入上传队列 |
| 文件超过大小限制 | 同上，显示「文件超过 500MB 限制」 |
| 网络中断 | 上传失败标红，显示「重试」按钮，支持断点续传（后续迭代） |
| 重名文件 | 提示「已存在同名文件」，提供「覆盖 / 保留两者 / 取消」三选项 |
| 列表为空 | 空状态插画 + 「立即上传文件」CTA 按钮 |
| 无搜索结果 | 空状态 + 「清除筛选条件」链接 |

## 七、非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 列表首屏（100 条）加载 ≤ 1s；文件列表支持虚拟滚动（万级数据） |
| 安全 | 文件上传前进行病毒扫描（集成云存储安全能力）；访问需登录鉴权 |
| 可用性 | 键盘可访问（Tab / Enter / Esc / Delete）；符合 WCAG 2.1 AA |
| 国际化 | 界面支持中/英文切换；文件名支持全字符集 |

## 八、后续迭代规划（Out of Scope for v1）

1. **分享与权限控制**：生成链接，设置有效期与密码
2. **版本管理**：同名文件自动保留历史版本
3. **OCR / AI 解析**：上传后自动提取文本内容，可供 AI 检索
4. **断点续传**：大文件中断后可从断点继续
5. **第三方导入**：对接 Google Drive / Dropbox / Notion
6. **协作共享专辑**：多人共同维护同一专辑
