# 新手首页体验 PRD

> 版本：v2.0 | 状态：已实现 | 日期：2026-02-28

---

## 一、背景与目标

### 1.1 问题定义

现有首页面向的是已沉淀了任务、知识库、定时任务的**熟练用户**。新用户首次进入时会面临：

- 右侧栏出现「事件追踪 / 报告进度 / 协作进度」，但内容是虚假占位数据，制造错误预期
- 打招呼话术预设了「已有历史任务」的语境（"今天会帮你完成本周周报"）
- 没有任何引导，用户不知道该做什么
- MagicInput 功能入口太多，新手无从下手
- 探索精选案例仅供浏览，无法跟新手行程「第一件事」的联动

### 1.2 目标

| 目标 | 指标 |
|------|------|
| 降低首次操作摩擦 | 新用户 5 分钟内完成第一个任务的比率 ≥ 60% |
| 提升留存 | 第 3 日留存率相对提升 20% |
| 增强个性化 | 用户完成自我介绍的比率 ≥ 40% |
| 渐进披露功能 | 用户感知「功能太多太复杂」的负反馈减少 50% |

## 二、新手状态定义与触发时机

### 2.1 状态机

```
NEW_USER → INTRODUCED → FIRST_TASK_STARTED → FIRST_TASK_DONE → GROWING
```

| 阶段 | 标识 | 触发条件 |
|------|------|----------|
| `NEW_USER` | 进入时无任何历史任务、知识库文件、定时任务 | 默认状态 |
| `INTRODUCED` | 用户完成了自我介绍（至少填写职业/行业） | 提交自我介绍表单 |
| `FIRST_TASK_STARTED` | 用户第一次触发了 Chat / 任务执行 | onOpenChat() 调用 |
| `FIRST_TASK_DONE` | AI 完成了第一次对话/任务 | Chat 关闭或 AI 返回最终结果 |
| `GROWING` | 完成 3 个不同类型任务，或手动切换为专家模式 | 任务计数器或模式切换 |
```

> `appMode === 'beginner'` 只要为 `true`，就渲染新手视图；切换为 `'normal'` 则完整渲染专家首页。

### 2.2 本地持久化

使用 `localStorage` 存储新手进度，key 为 `tbox_onboarding_state`：

```ts
interface OnboardingState {
  stage: 'new_user' | 'introduced' | 'first_task_started' | 'first_task_done' | 'growing';
  profile?: UserProfile;       // 自我介绍数据
  completedTaskTypes: string[]; // 已完成的任务类型
  firstSeenAt: string;          // ISO 日期
  graduatedAt?: string;         // 自动毕业时间
}
```

## 三、整体布局差异（新手 vs 专家）

| 区域 | 专家模式 | 新手模式 |
|------|----------|----------|
| **右侧栏** | 显示（事件追踪 / 报告进度 / 协作进度） | **完全隐藏**，不占据任何空间 |
| **左侧栏内容** | 话题空间、档案库、集市、定时任务、历史对话 | **与专家模式相同，全量显示**；新手引导聚焦在中央区域，而非锁住侧栏 |
| **Center 顶部问候** | 基于历史任务的个性化回顾 | 首次见面式欢迎语 + 自我介绍触发器 |
| **Center 主区域** | 待确认事项列表 + MagicInput + 探索案例 | 聚焦卡片式引导区 + 简化输入框 |
| **背景** | 当前风景背景 | 轻柔渐变，视觉更干净，不分散注意力 |

## 四、新手首页分区详细设计

### 4.1 顶部问候区（Hero Greeting）

#### 复制文案策略

**未完成自我介绍时：**

```
（图标）  嗨，我是 Tbox，你的 AI 工作伙伴。
         很高兴认识你！先聊几句，我会更了解你 →
```

**完成自我介绍后（已知姓名 & 职业）：**

```
（图标）  嗨 Lisa，欢迎来到 Tbox！
         作为一位{职业}，我已经准备好帮你处理日常琐事了。
         来尝试一件事吧 ↓
```

#### 自我介绍触发器（ProfileIntroCard）

- 外观：一张轻柔卡片，用 `border-dashed` 虚线边框区别于普通内容卡
- 包含 3 个轻量字段（不强迫，随时可跳过）：
  1. **你的称呼** — 问句形式：`Lisa，你希望我怎么称呼你？`（Lisa 为系统已知的用户名）；placeholder 为「就叫 Lisa 吧，或者告诉我你的昵称」
  2. **你的职业 / 行业** — 下拉选择（产品经理 / 运营 / 销售 / 创业者 / 研究员 / 学生 / 其他）
  3. **你最常用来干嘛** — 多选 Tag（写方案 / 做报告 / 写代码 / 搜信息 / 发邮件 / 管项目 / 做PPT / 数据分析）
- **联动效果**：每次切换「主要用来做什么」Tag，下方任务快启区实时更新排序，与用途匹配的场景卡前置并显示「推荐」角标
- CTA：「开始体验 →」按钮，填写后保存至 localStorage 并进入 `INTRODUCED` 阶段
- 跳过链接：「先看看再说」，点击直接进入下方引导区，不再显示此卡

### 4.2 新手任务快启区（Onboarding Task Launcher）

**原则**：用 3 步内完成"第一件事"，不让用户面对空白输入框发愁。

#### 布局结构

```
┌─────────────────────────────────────────┐
│  你想先做哪一件事？                      │
│  ──────────────────────────────────      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 📊 写报告  │ │ 🔍 搜信息  │ │ 📧 写邮件  │ │
│  │ 市场调研   │ │ 竞品分析   │ │ 商务洽谈   │ │
│  └──────────┘ └──────────┘ └──────────┘ │
│  ┌──────────┐ ┌──────────┐              │
│  │ 📋 做方案  │ │ ✨ 随便聊聊 │              │
│  │ 推广计划   │ │ 随便说什么  │              │
│  └──────────┘ └──────────┘              │
└─────────────────────────────────────────┘
```

- 每张卡片点击后，输入框聚焦，并在下方展示对应场景的**输入联想提示**（见 4.3）
- 已选场景卡片高亮，其余轻微淡化（专注引导）
- 每个场景内置 3 条"快速预填示例"，也可自由编辑

#### 新手任务卡片配置

| 卡片 | 图标 | 口语化标签 | 描述 | 对应 Mode |
|------|------|------------|------|--------|
| 写报告（Research） | 📊 | 写个报告 | 市场调研、竞品分析都行 | Web 搜索 → 深度研究 |
| 搜信息（Search） | 🔍 | 帮我搜搜 | 热点资讯、数据查询随时找 | Web 搜索 → 实时热点 |
| 做 PPT（Present） | 📑 | 整个PPT | 路演汇报、培训课件都能做 | PPT 演示 → 路演报告 |
| 写邮件（Email） | 📧 | 写封邮件 | 商务沟通、客诉跟进都顺手 | Mail 撰写 → 商务洽谈 |
| 做方案（Plan） | 📋 | 搞个方案 | 推广活动、运营计划都来吧 | ChatGPT 自由模式 |
| 随便聊聊（Chat） | 💬 | 随便聊聊 | 啥都能问，不用客气 | 对话 |

**用途 → 场景映射（用于动态排序）**

| 用途 Tag | 优先场景 ID |
|----------|-----------|
| 写方案 | plan |
| 做报告 | research |
| 写代码 | chat |
| 搜信息 | search |
| 发邮件 | email |
| 管项目 | plan |
| 做 PPT | ppt |
| 数据分析 | research |

当用户在 ProfileIntroCard 选择了用途 Tag 时，TaskLauncher 实时重新排序：匹配的场景卡前置，右上角显示「推荐」角标，标题改为「根据你的目标，推荐先试试：」。

### 4.3 智能输入框（Smart Input）

新手模式下，MagicInput 做如下调整：

#### A. 精简工具栏

- 只露出最常用的 3 个 Mode 图标（PPT / 搜索 / 邮件）
- 右侧有「更多」展开按钮，点击才展示全部
- 不展示 Agents / Skills 等高级选项（避免信息过载）

#### B. 输入联想（Input Suggestion Chips）

当用户选定某个场景卡片后，输入框上方出现联想提示条，格式：

```
💡 你可以这样说：
   「帮我调研2026年新能源汽车市场趋势」
   「整理一下目前国内AI编程助手的竞品对比」
   「写一份关于代餐品牌推广的市场分析报告」
```

- 3 条提示为 Chip 按钮，点击直接填入输入框（可继续编辑）
- 提示内容根据「职业 / 行业」个性化匹配（若完成了自我介绍）
- 用户开始自由输入时，提示区域平滑收起

#### C. 占位符（Placeholder）轮播

当输入框为空、且未选择场景时，placeholder 轮播展示：

```
试试：「帮我写一份 Q2 产品规划方案」
试试：「搜索最近一周 AI 领域的重要新闻」
试试：「帮我起草一封与甲方续约的商务邮件」
试试：「整理一下竞品 A 和 B 的核心差异」
```

每 3 秒切换一条，切换时有淡入淡出动画。

### 4.4 进度感知区（Onboarding Progress）

**位置**：任务快启区下方，始终可见（不需滚动）。

**形态**：一条水平进度卡，3 个步骤：

```
● 认识一下（已完成）  ──  ○ 完成第一件事  ──  ○ 解锁更多玩法
```

- 当前正在执行的步骤显示阶段描述文字
- 完成一步后，对应节点变为绿色打钩，并有庆祝微动画
- 整体进度条填充（绿色渐进）

---

### 4.5 第一次任务完成后的「解锁提示」

**触发时机**：`FIRST_TASK_DONE` 状态

**形式**：在进度卡下方弹出一张 Banner 卡（可关闭）：

```
🎉 太棒了！你完成了第一件事。
   发现更多 Tbox 的能力：
   ┌──────────────────────┐  ┌──────────────────────┐
   │ 🗓 设置定时任务        │  │ 🧠 上传你的文件档案     │
   │ 让 AI 每天自动为你工作 │  │ 让 AI 理解你的业务背景  │
   └──────────────────────┘  └──────────────────────┘
   ┌──────────────────────┐  ┌──────────────────────┐
   │ 🛒 逛逛集市 Agent      │  │ 🔁 试试另一种任务类型   │
   │ 找专属 AI 助手         │  │ 还有 5 种能力等你探索   │
   └──────────────────────┘  └──────────────────────┘
```

- 点击集市、定时任务、档案库卡片，直接跳转到对应视图
- 「我知道了，继续使用」关闭此 Banner，不再显示

---

### 4.6 新手左侧栏（Simplified Sidebar）

新手模式下，左侧栏**与专家模式完全相同，全量显示所有功能入口**。不做任何锁定或隐藏。

理由：侧栏中的功能条目不是新手的注意力瓶颈；真正需要引导的内容在中央区域。减少功能项反而会让用户感到平台「很简陋」，增大流失风险。

**完成第一次任务后的引导**（UnlockBanner）出现在中央区域，引导用户探索 4 个延伸方向：
- 🗓 设置定时任务 → 点击跳转 scheduled 视图
- 🧠 上传档案 → 点击跳转 knowledge 视图
- 🛒 逛逛集市 Agent → 点击跳转 market 视图
- 🔁 试试另一种任务 → 清空选择，重新选场景

## 五、视觉设计原则（新手模式专属）

### 5.1 聚焦策略

- **最多同屏 2 个 CTA**，避免行动选项过多造成决策疲劳
- 当用户选中某个任务卡片后，其他卡片**轻度虚化**（`opacity: 0.4`），聚焦当前选项
- 背景图渐变降噪：从当前风景图切换为**纯色/轻渐变**背景（米白色 → 浅灰，更干净）

### 5.2 排版层级

```
Level 1: 主标题 / 用户称呼   → 24px, font-semibold
Level 2: 区域标题             → 13px, font-semibold, text-[#6e6e73]
Level 3: 卡片标题             → 15px, font-medium
Level 4: 辅助文字/说明         → 13px, text-[#86868b]
```

### 5.3 色彩规范

| 用途 | 颜色 |
|------|------|
| 新手高亮 CTA | `#2563eb`（蓝色，区别于熟练用户的黑色） |
| 进度完成 | `#22c55e`（绿色） |
| 进度未完成 | `#e5e7eb`（浅灰） |
| 场景卡片选中 | `rgba(37,99,235,0.08)` 背景 + 蓝色边框 |
| 解锁提示 Banner | `rgba(255,255,255,0.9)` + 1px 蓝色边框 |

### 5.4 动效规范

| 交互 | 动效 |
|------|------|
| 场景卡片点击 | scale(0.97) 按压感 + border-color 变化，150ms ease |
| 输入联想出现 | translateY(-8px) → 0 + opacity 0→1，200ms ease-out |
| 进度步骤完成 | 打钩图标 scale(0→1.2→1)，填充色渐变，300ms |
| 解锁 Banner 出现 | slide-up + fade-in，250ms ease-out |
| 占位符轮播 | 竖向淡入淡出，300ms |

## 九、边界情况处理

| 情况 | 处理方案 |
|------|----------|
| 用户刷新页面 | 从 localStorage 读取 onboardingState，恢复对应阶段 |
| 用户在聊天中途关闭页面 | `taskStarted` 已记录，下次进入仍显示"继续你上次的任务" |
| 用户立刻切换到专家模式 | 直接跳过新手引导，所有功能全开，清除 beginner 状态 |
| 第一次任务执行失败 | 不推进阶段，显示"换个说法再试试" + 3 条重新联想 Chip |
| 用户已是老用户手动切换到新手模式 | 展示新手视图，但阶段从 `INTRODUCED` 开始（已知用户，不用再介绍） |
| 清除浏览器缓存 | 重置为全新状态，重新走完整新手流程 |
