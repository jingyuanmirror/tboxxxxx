# 话题空间（Project Space）PRD

> 版本：v0.2 · 日期：2026-03-03 · 状态：草稿

---

## 一、背景与目标

### 1.1 背景

当前产品左侧边栏已有"话题空间"导航入口，展示若干项目卡片（如"Q4 产品推广方案"、"芯片市场竞品分析"）。但点击后尚无专属的项目空间页面，用户的文档、对话、Agent 协作、任务等均分散在不同模块中，缺乏以"项目"为核心的统一工作上下文。

### 1.2 目标

打造以**项目为中心**的协作空间，让用户在一个统一界面内完成：
- 项目文档的沉淀与管理
- 与专属 Agent 团队的持续对话
- 历史对话记录的归档与检索
- 项目成员（Agent + 真人）的管理
- 任务派发、进度追踪与成果汇聚

**核心价值主张**：进入话题空间 = 进入一个完整的项目工作室，所有上下文都在这里，随时继续、随时回溯。

## 二、用户故事

| # | As a... | I want to... | So that... |
|---|---------|-------------|-----------|
| U1 | 项目负责人 | 进入某个话题空间后，看到该项目所有文档（我上传的 + Agent 产出的） | 不需要在档案库里翻找 |
| U2 | 项目负责人 | 查看与本项目相关的全部历史对话 | 随时回顾决策上下文 |
| U3 | 项目负责人 | 看到并管理为该项目雇佣的 Agent 成员 | 知道谁在干什么 |
| U4 | 项目负责人 | 在空间内直接向 Agent 发起对话，并继承项目上下文 | 无需重新说明背景 |
| U5 | 项目协作者 | 看到项目当前进展、各 Agent 产出的结果 | 掌握全局进度 |
| U6 | 项目负责人 | 为空间设定目标与里程碑 | 让 Agent 知道边界与优先级 |
| U7 | 项目负责人 | 快速向空间发布任务、指定负责 Agent | 直接派单而不跳转到任务市场 |
| U8 | 用户 | 在话题空间内既能和 Tbox 对话，也能和我加入的 Agent 对话 | 根据需要灵活选择对话对象，各取所长 |
| U9 | 用户 | 给项目 Tbox 起一个专项名字（如"Q4 策划师"），使其成为我的新 Agent | 形成专门服务这个项目的定制化助手，可在其他场景复用 |
| U10 | 用户 | 给 Tbox 添加技能（Skill），让它拥有专项能力 | 技能在左侧技能栏显示，随时查看和管理 |
| U11 | 用户 | 新建话题空间时，Tbox 主动询问是否给它起专项名 | 在项目起始阶段就建立个性化的 AI 工作伙伴 |
| U12 | 用户 | 新建对话时默认和 Tbox 对话，也可切换到其他 Agent | 无需额外选择即可快速开始，同时保留灵活性 |

## 三、功能架构

```
话题空间（TopicSpace）
├── 空间头部（SpaceHeader）
│   ├── 项目封面 / Emoji
│   ├── 项目名 + 描述（可编辑）
│   ├── 项目标签 & 状态
│   ├── 空间设置入口
│   └── 成员头像列表（快速预览）
│
├── 项目 Tbox（ProjectTbox）                 ← 每个空间独享的 Tbox 实例
│   ├── 继承项目上下文（目标 / 文档 / 成员）
│   ├── 可自定义名字（命名后成为新 Agent）
│   ├── 可添加技能（Skill）→ 同步显示到左侧技能栏
│   └── 所有对话默认与 Tbox 开始
│
├── 顶部 Tab 导航
│   ├── 概览（Overview）
│   ├── 文档（Documents）
│   ├── 对话（Conversations）          ← 分 Tbox 对话 & Agent 对话两类
│   ├── 成员（Team）                   ← 含 Tbox + 受邀 Agent
│   ├── 任务（Tasks）
│   └── 设置（Settings）
│
└── 浮动对话框（FloatingChat）
    └── 随时唤起，默认与 Tbox 对话，可切换对话对象
```

## 四、各模块详细设计

### 4.1 空间头部（SpaceHeader）

**展示内容：**
- 项目封面图（可自定义，默认随机渐变色块）
- 项目 Emoji / 图标（16 个预设 + 自定义上传）
- 项目名称（单击可编辑，最多 40 字）
- 一句话描述（可选，最多 100 字）
- 标签组（最多 5 个，颜色可选）
- 项目状态徽章：`进行中` / `已完成` / `暂停中` / `计划中`
- 成员头像列表（最多展示 5 个，超出显示 +N）
- 创建时间 & 最后活跃时间

**交互：**
- 点击封面区域 → 弹出封面选择器（纯色 / 渐变 / 上传图片）
- 点击名称 → inline 编辑
- 点击状态徽章 → 下拉更换状态

---

### 4.2 概览页（Overview）

概览是进入空间后的默认落地页，提供项目的全局视角。

**布局：** 左主区（2/3）+ 右侧信息栏（1/3）

**左主区：**

| 区块 | 内容 |
|------|------|
| **最近活动流** | 时间线形式展示：最新对话摘要、新增文档、任务状态变更、Agent 产出成果等 |
| **待处理事项** | 需要用户确认 / 回复的 Agent 消息、新增未读文档、进行中的任务快照 |
| **快速启动区** | 3 个常用操作按钮（发起对话 / 上传文档 / 发布任务）|

**右侧信息栏：**

| 区块 | 内容 |
|------|------|
| **项目目标** | 用户设定的目标描述，可 Markdown，Agent 能读取以保持上下文对齐 |
| **里程碑** | 时间节点 + 完成状态，纵向时间线 |
| **成员速览** | Agent 卡片列表（头像 + 角色 + 最近活跃时间）|
| **关键数据** | 文档数 / 对话轮次 / 任务完成率 / 用量消耗 |

---

### 4.3 文档页（Documents）

项目内所有文档的统一管理，区分**我的文档**与 **Agent 产出**两大来源。Agent 产出的文档无需审核确认，产出即可查阅，无需用户二次确认。

**布局：** 顶部来源筛选栏 + 标签筛选栏 + 文档卡片网格

**顶部来源筛选栏（三项）：**
`全部文档` / `我上传的` / `Agent 产出` — 切换后同时刷新文档列表和可用标签

**标签筛选栏（动态）：**
- 来源切换后，动态汇总当前范围内所有文档的标签
- 标签以圆形胶囊展示；点击激活（coral 高亮），再次点击取消
- 标签与来源过滤取交集

**文档卡片：**
- 格式徽章（颜色编码：PDF / DOCX / XLSX / PPTX / MD / PNG）
- 文档标题
- Agent 头像 + 名称（Agent 产出时）
- **来源对话**：Agent 产出文档显示生成该文档的对话名称，带 MessageSquare 图标
- 文档标签（浅灰圆形胶囊）
- NEW 角标（仅 `isNew === true` 时显示）
- 文件大小 · 创建时间

**设计说明：**
- 不使用文件夹层级，改为标签扁平筛选，降低信息组织复杂度
- 每份文档可携带多个标签（如"方案""竞品""分析"），由用户或 Agent 在产出时标注

**文件夹（Folder）：**
- 每个空间可配置若干文件夹，用于将文档分组管理（如「策略文档」「设计物料」「数据报告」）
- 文档创建时可指定 `folderId`，无 folderId 的文档属于「未分类」
- 选中文件夹胶囊即可筛选该文件夹下的文档

**未读（NEW）机制：**
- 每份文档携带 `isNew: boolean` 字段，表示用户是否已查看过该文档
- 左侧导航栏「文档」tab 上的数字徽章 = 空间内 `isNew === true` 的文档数量
- 文档卡片右上角展示显眼的 `NEW` 角标，提示尚未查阅
- 概览页「新文档」区块展示所有 `isNew === true` 的文档列表
- 点击文档卡片进入查看后，该文档 `isNew` 置为 `false`，徽章计数同步减少

**文档卡片信息：**
- 文件格式徽章（颜色编码：PDF / DOCX / XLSX / PPT / MD 等）
- 文档标题
- 来源信息：Agent 头像 + 名称（Agent 产出）或「我上传的」
- 元信息：文件大小、创建时间
- NEW 角标（仅 `isNew === true` 时展示）

**主内容区操作：**
- 排序：时间 / 类型 / 来源
- 搜索：全文检索（本空间范围内）
- 批量操作：移动到文件夹 / 删除 / 下载

### 4.4 对话页（Conversations）

项目内全部对话的归档与入口，实现**历史可查、上下文可接续**。对话分为两类：**Tbox 对话**（与项目 Tbox 的对话）和 **Agent 对话**（与受邀 Agent 成员的对话）。

**布局：** 左侧会话列表 + 右侧对话内容区

**会话列表（左侧）：**
- 固定在顶部：`+ 新建对话` 按钮
  - 点击后弹出对话目标选择器，**默认高亮选中 Tbox**
  - 可切换到任意受邀 Agent
- 搜索框（搜索对话主题 / 内容关键词）
- 对话分组：
  - **与 Tbox 的对话**（🤖 Tbox 图标，显示 Tbox 名字）
  - **与 Agent 的对话**（分 Agent 展示各自的对话列表）
- 每条会话卡片：
  - 对话对象头像（Tbox 或 Agent）
  - 会话标题（自动摘要，可手动编辑）
  - 最后一条消息预览
  - 时间戳
  - 产出文档数量标记

**对话内容区（右侧）：**
- 顶部展示对话对象信息：头像 + 名称 + 角色（Tbox / Agent 职能）
- 顶部额外展示：**项目上下文注入状态**（绿色 = 已注入项目背景、目标、相关文档）
- 完整的 ChatDialog 界面
- 对话过程中 Agent / Tbox 产出的文档会自动关联到本空间文档页
- 消息操作：点赞 / 引用 / 导出为文档 / 标记为里程碑
- 底部输入框：支持 `@成员` 唤起特定 Agent，支持 `/命令` 快捷操作

**新建对话目标选择器（弹窗）：**

```
┌─────────────────────────────────────┐
│  选择对话对象                         │
│                                     │
│  ┌──────────────────────────────┐   │
│  │ 🤖 [Tbox 名字]  ← 默认选中   │   │
│  │    你的项目助手 · 已注入项目上下文│  │
│  └──────────────────────────────┘   │
│                                     │
│  Agent 成员                          │
│  ○ Aria — 营销策划                   │
│  ○ Sage — 数据分析                   │
│  ○ Nova — 视觉设计                   │
│                                     │
│              [ 开始对话 ]             │
└─────────────────────────────────────┘
```

**项目上下文注入机制：**

当用户在空间内发起对话时，系统自动向 Agent / Tbox 注入以下背景：
```
[项目名称] + [项目描述] + [当前目标] + [近期里程碑] + [关联文档摘要（可选）]
```
用户可在会话设置中选择注入哪些上下文。

### 4.5 成员页（Team）

管理为本项目服务的 Agent 成员（含 **项目 Tbox**、平台 Agent 和用户自定义 Agent）。

**布局：** Tbox 置顶卡片 + Agent 成员卡片网格

**Tbox 置顶卡片（特殊样式）：**
- 占据整行，视觉上区别于普通成员卡片
- 展示：头像 + 名字（含"项目 Tbox"副标签）+ 已添加技能列表（最多 3 个 Skill 标签，超出显示 +N）
- 操作入口：`为 Tbox 添加技能` / `给 Tbox 起名` / `发起对话`
- 若已命名 → 显示自定义名字和"📌 已保存为我的 Agent"徽章

**普通成员卡片信息：**
- Agent 头像 + 名称 + 标签（来自市场 / 自建）
- 角色/职能描述（如：研究员 / 内容创作者 / 数据分析师）
- 状态：`工作中` / `空闲` / `待命`
- 本项目内统计：完成任务数 / 产出文档数 / 对话轮次
- 操作：发起对话 / 查看历史 / 解除雇佣 / 调整权限

**邀请成员：**
- 入口：右上角 `+ 邀请成员` 按钮
- 方式一：从"市场"快速搜索并雇佣 Agent
- 方式二：从"我的清单"调取已雇佣的 Agent
- 方式三：邀请真人协作者（输入邮箱，后续功能）

**权限设计（当前版本简化）：**
- `Tbox`：最高权限，可读写所有文档，可自主发起子任务，拥有项目完整上下文
- `负责 Agent`：可读写所有文档，可自主发起子任务
- `协作 Agent`：可读文档，可写分配给自己的区域
- `观察者`：只读

### 4.6 任务页（Tasks）

在空间内直接管理与本项目相关的所有任务，无需跳转任务市场。

**布局：** 看板视图（Kanban）/ 列表视图切换

**看板列：**
```
📥 待分配 → 🔄 进行中 → 👀 待审核 → ✅ 已完成
```

**任务卡片信息：**
- 任务标题 + 类型图标
- 负责 Agent 头像
- 截止时间 + 优先级标签
- 预算区间（如有）
- 产出文档数量
- 快速操作：催进度 / 追加说明 / 审核通过

**新建任务：**
- 支持**自然语言创建**：在空间对话框内输入 `/task 帮我分析竞品` → 自动生成任务卡片
- 支持表单创建：标题 / 描述 / 截止时间 / 负责 Agent / 预算 / 优先级
- 创建后可选择"立即派单到市场"或"仅分配给成员"

### 4.7 项目 Tbox（ProjectTbox）

> **核心定位**：每个话题空间拥有一个独属的 Tbox 实例，它是该项目的"全知助手"——自动继承项目上下文，可被个性化命名成为用户的新 Agent，并通过添加技能扩展能力边界。

#### 4.7.1 基础能力

| 能力 | 说明 |
|------|------|
| **项目上下文感知** | 自动加载项目名称、目标、里程碑、空间文档摘要，对话无需每次重新说明背景 |
| **默认对话入口** | 新建对话时默认与 Tbox 开始，也可切换到其他 Agent |
| **文档产出关联** | Tbox 对话产出的文档自动归档到本空间文档页，携带 `conversationId` |
| **任务派发** | 可从对话中 `/task` 快捷派单给空间内的 Agent 成员 |

#### 4.7.2 Skill（技能）系统

用户可为项目 Tbox 添加技能，使其获得专项执行能力。

**技能的呈现：**
- 左侧边栏"我的技能"区域 → 显示已给 Tbox 及其他 Agent 添加的技能列表
- Skill 标签显示在 Tbox 的成员页卡片上（最多 3 个 + 更多）
- 对话输入框中输入 `/技能名` 可直接调用该技能

**技能的来源：**
1. **系统预置 Skill**：搜索引擎 / 图像生成 / 代码执行 / 文件解析 / 日历读写…
2. **从装备铺购买**：第三方 Skill 插件，购买后可分配给指定 Agent / Tbox
3. **自定义 Skill**：用户自己通过 API 配置的自定义 Skill

**添加 Skill 流程：**
```
成员页 Tbox 卡片 → 点击「添加技能」
  → 弹出 Skill 选择面板（分类浏览 / 搜索）
  → 选中后确认 → Skill 即时生效
  → 左侧边栏「我的技能」新增该 Skill 条目
  → Tbox 对话中可通过 /命令 调用
```

#### 4.7.3 命名为新 Agent（个性化定制）

用户可以为项目 Tbox 起一个专属名字，将其"升格"为一个完整的新 Agent，复用于其他场景。

**命名流程（成员页触发）：**
```
点击 Tbox 卡片上的「给 Tbox 起名」
  → 弹出命名对话框
  → 输入：自定义名字（如"Q4 策划师"）
  → 可选：设置头像（预设 16 个 + 上传）、一句话介绍
  → 确认后
      ├── Tbox 以新名字显示（全局生效）
      ├── 该 Agent 出现在「我的清单」内
      └── 右下角 Toast：「已保存为新 Agent，可在其他话题空间使用」
```

**命名状态展示：**
- 已命名：头像旁显示自定义名字 + `📌 我的 Agent` 标签
- 未命名：显示默认"Tbox · [项目名]"

#### 4.7.4 新建空间时的 Tbox 命名引导

新建话题空间流程结束后，Tbox 主动在对话区发送一条引导消息：

> **对话气泡示例：**
> ```
> 🤖 Tbox
> ─────────────────────────────────────
> 嗨！我是你在「Q4 产品推广方案」的专属助手。
> 要不要给我起个专项的名字？有了名字，我就能
> 成为你的独立 Agent，在其他项目里也能找到我。
>
> [✏️ 给我起个名字]   [暂时不了]
> ─────────────────────────────────────
> ```

- 点击「给我起个名字」→ 直接进入命名对话框（§4.7.3）
- 点击「暂时不了」→ 关闭引导，不再重复弹出（可在成员页重新触发）

### 4.8 设置页（Settings）

**项目基础信息：**
- 名称 / 描述 / 封面 / 标签 / 状态

**项目 Tbox 设置：**
- Tbox 名字（可修改）
- Tbox 头像（可修改）
- 已添加的技能列表（可删除某项技能）
- Tbox 默认注入的上下文类型（项目描述 / 目标 / 里程碑 / 相关文档）

**通知设置：**
- Agent 产出新文档时通知
- 任务状态变化时通知
- Agent 需要用户确认时通知

**危险操作：**
- 归档空间 / 删除空间

### 4.9 浮动对话框（FloatingChat）

在空间内任意页面（除对话页外），右下角悬浮一个 Chat 气泡按钮，点击后弹出抽屉式对话框，**不打断当前工作上下文**。

**特性：**
- 小屏模式：不遮挡主内容，侧边滑出（宽 400px）
- **默认与 Tbox 对话**；右上角切换按钮可切换到其他 Agent
- 自动携带项目上下文注入
- 支持 `@` 指定 Agent
- 消息可一键"保存到文档"或"创建为任务"
- 对话结束后自动归档到对话页

## 五、信息架构与数据模型（前端 Mock 阶段）

### 5.1 TopicSpace

```typescript
interface TopicSpace {
  id: string;
```

## 六、交互流程

### 6.1 进入空间

```
左侧边栏点击话题空间项目名
  → 主内容区切换到 TopicSpace 视图（全屏，覆盖 CenterMain 原有内容）
  → 默认展示 Overview 页
  → 右侧边栏隐藏（空间已有全量信息）
```

### 6.2 新建空间

```
左侧边栏"话题空间"区域点击 + 号
  → 弹出创建弹窗：输入项目名 → 选择 Emoji → 可选添加描述
  → 确认后进入空间，默认展示对话页（Tbox 对话）
  → Tbox 自动发送命名引导气泡（§4.7.4）
  → 引导步骤：
      Step 1: 给 Tbox 起名（或跳过）
      Step 2: 邀请 Agent 成员
      Step 3: 设定项目目标
      Step 4: 上传第一份文档 或 发起第一次对话
```

### 6.3 新建对话

```
点击「+ 新建对话」按钮
  → 弹出对话目标选择器
  → 默认高亮选中「Tbox（自定义名 / 默认名）」
  → 用户可选择切换到受邀 Agent
  → 点击「开始对话」→ 进入对话内容区
  → 系统自动注入项目上下文，顶部显示绿色注入状态
```

### 6.4 给 Tbox 添加 Skill

```
成员页 → Tbox 置顶卡片 → 点击「添加技能」
  或 设置页 → 项目 Tbox 设置 → 技能管理
  → 弹出 Skill 选择面板（分类 + 搜索）
  → 选中某 Skill → 预览说明 → 确认添加
  → Skill 即时生效：
      ├── Tbox 对话输入框提示新的 /命令
      ├── 成员页 Tbox 卡片更新 Skill 标签
      └── 左侧边栏「我的技能」新增条目
```

### 6.5 Tbox 命名为新 Agent

```
成员页 Tbox 卡片 → 点击「给 Tbox 起名」
  或 Tbox 命名引导气泡 → 点击「给我起个名字」
  → 弹出命名对话框
  → 输入名字 + 可选设置头像 / 介绍
  → 确认
      ├── 全局标记 savedAsAgent = true
      ├── 在「我的清单」中创建该 Agent 条目
      ├── 成员页 Tbox 卡片更新为自定义名字 + 📌 徽章
      └── Toast：「已保存为我的 Agent」
```

### 6.6 Agent 产出文档审核

```
Agent 对话过程中生成文档
  → 自动出现在文档页"待审核"分类 + 概览页"待处理事项"
  → 用户点击文档卡片 → 预览全文
  → 操作：通过 / 需修正（输入修正指令，直接发回给 Agent）
  → Agent 接收修正指令后重新生成，再次进入待审核流程
```

### 6.7 快速发布任务（自然语言）

```
浮动对话框 或 对话页，输入：/task [任务描述]
  → 系统解析意图，展示任务表单预填（标题/描述/优先级/截止时间估算）
  → 用户确认 / 修改
  → 创建任务，出现在任务页
  → 可选：立即分配给某位成员 Agent
```

## 七、UI 设计规范

### 7.1 视觉风格

- 延续现有产品的 macOS 设计语言：浅色背景、大圆角、玻璃拟态
- 空间封面区采用全宽大图，形成沉浸感
- Tab 导航采用胶囊形 active 状态，与现有 MarketView 保持一致
- 文档卡片、成员卡片参考现有 KnowledgeView / MarketView 的卡片规范

### 7.2 颜色系统

| 用途 | 颜色 |
|------|------|
| Agent 产出标记 | `#f4845f`（现有品牌橙色） |
| 待审核状态 | `#f59e0b` 琥珀色 |
| 已通过状态 | `#22c55e` 绿色 |
| 需修正状态 | `#ef4444` 红色 |
| 空间 active 背景 | `rgba(255,255,255,0.6)` |

### 7.3 响应式

- 桌面（≥ 1280px）：左侧文档树 + 主内容区双栏
- 平板（768px ~ 1279px）：侧边树折叠为顶部 Tab
- 移动端：全屏单列，浮动对话框底部抽屉

## 九、待讨论与开放问题

| # | 问题 | 建议方向 |
|---|------|---------|
| Q1 | 话题空间与"档案库"的文档如何双向同步？ | 档案库为全局视图，空间为项目视图，同一文档多处引用而非复制 |
| Q2 | 多个 Agent 同时进行对话时如何管理上下文隔离？ | 每个会话独立维护上下文，成员页可查看各 Agent 当前激活会话 |
| Q3 | 空间级别的通知如何整合到全局 Header？ | Header 增加铃铛图标，分组展示各空间的待处理通知 |
| Q4 | 是否支持空间内的实时协作（多用户）？ | Phase 1 单用户，Phase 5 后引入协作者邀请 |
| Q5 | Agent 产出文档的版本管理策略？ | 每次修正保留历史版本，侧边展示版本轴 |
| Q6 | Tbox 命名后，多个空间的 Tbox 如果同名会产生混淆吗？ | 每个 Tbox 有唯一 agentId，名字仅作为显示标签，允许重名；"我的清单"中以「名字 · 来源空间」区分 |
| Q7 | Skill 是否跨空间共享还是每个 Tbox 独立？ | Skill 分配给具体的 Tbox 实例，各空间 Tbox 独立持有；但用户可在多个 Tbox 上叠加同一 Skill |
| Q8 | Tbox 的 Skill 与受邀 Agent 的内置能力如何在对话中协同？ | 用户在对话中可 @ 特定 Agent 触发其能力；Tbox 负责统筹和分发，Agent 负责专项执行 |
| Q9 | 用户已将某 Tbox 命名为 Agent 后，删除该空间，Agent 是否保留？ | 保留，Agent 独立存在于"我的清单"，与源空间解绑但标注「原始空间已归档」 |
